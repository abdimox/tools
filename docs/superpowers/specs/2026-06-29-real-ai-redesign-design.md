# 乐活互动小红书 AI 运营工作台真实 AI 改版规格

## 背景

当前版本使用本地 `DemoProvider` 输出规则模板，不能真正理解活动场景、账号截图和同行内容。本次改版删除演示结果，接入 `https://llmhub.ltd` 中转站。统一使用 `gpt-5.5` 处理文本与视觉理解，使用 `gpt-image-2` 处理封面图片编辑。

本次改版的评价标准不是生成数量，而是内容能否提高点击意愿、建立专业信任并带来更精准的合作咨询。

## 已选技术方案

采用一个中转站配置、两个模型、真实接口失败时不降级到模板的方案。

- API Base URL：默认 `https://llmhub.ltd/v1`，管理员可以修改。
- 文本与视觉模型：默认 `gpt-5.5`，管理员可以修改。
- 图片模型：默认 `gpt-image-2`，管理员可以修改。
- 文本、视觉和图片请求共享同一个中转站 API Key。
- 文本与视觉优先使用 OpenAI 兼容的 Chat Completions 接口。
- 图片生成使用 `/images/generations`，基于用户原图制作封面使用 `/images/edits`。
- 不保留 `DemoProvider` 作为用户可见降级路径。

如果中转站对模型使用自定义别名，管理员可在设置页直接修改模型字符串，无需改代码。

## 业务与场景

业务类型仍然必须先选，并严格隔离。

### 手作 DIY

场景为必选下拉框：

- 企业
- 商场
- 楼盘
- 社区
- 4S 店

### Photobooth

场景为必选下拉框：

- 婚礼
- 企业活动
- 宝宝宴
- 聚会

用户选择的场景是最高优先级事实。AI 不得改写、猜测或替换场景，只能从案例简介中补充城市、人数、项目和活动细节。

## 客户目的模型

系统提示词必须让模型先分析客户为什么举办活动，再开始写作。

- 企业：员工参与、团队氛围、流程可控、降低组织压力。
- 商场：吸引停留、亲子互动、节日氛围、提升场所好感。
- 楼盘：延长客户停留、创造自然交流机会、提升项目体验。
- 社区：邻里参与、低门槛互动、节日或公共文化体验。
- 4S 店：改善到店等待、增加家庭互动、提升品牌和展厅好感。
- 婚礼：宾客自然互动、现场纪念、减少等待和陌生感。
- 企业活动 Photobooth：签到打卡、现场互动、品牌或年会记忆点。
- 宝宝宴：家庭纪念、宾客参与、主题照片留存。
- 聚会：活跃气氛、降低互动门槛、形成可带走的照片记忆。

文案不得声称活动一定提升销售、成交或客流。只能结合真实机制说明其可能带来的体验价值。

## 工作流一：生成小红书笔记

笔记生成与图片完全分离，不允许上传图片。

输入：

1. 业务类型。
2. 对应业务的必选场景。
3. 一句案例简介。

输出：

- 3 个经过筛选的标题。
- 1 篇最终正文。
- 8–12 个话题，直接放在正文末尾。
- 一键复制完整正文和话题。
- 简洁的 AI 复核通过状态。

### 两阶段生成

第一次 `gpt-5.5` 调用负责客户意图分析与初稿，返回结构化 JSON。第二次调用作为独立编辑，接收原始输入和初稿，执行以下检查后返回最终 JSON：

- 场景是否与用户选择一致。
- 是否捏造用户没有提供的品牌、人数、效果、数据和现场细节。
- 标题是否包含搜索价值、场景信息和点击理由。
- 正文是否回答目标客户为什么需要这类活动。
- 手作 DIY 与 Photobooth 是否混写。
- 是否存在私信、微信、二维码、报价等导流风险。
- 话题是否与城市、业务、场景和项目相关。
- 是否符合 `humanizer-zh` 的人性化写作规则。

### Humanizer 规则

`humanizer-zh` 是开发期技能，部署后的网页不能直接调用 Codex Skill。系统会把规则整理进第二阶段编辑提示词：删除填充词、宣传腔、机械三段式、否定式排比、空泛结论、模糊归因、过度连接词和聊天机器人痕迹；保留具体信息，调整长短句节奏，并避免编造“真实感”。

最终结果必须通过结构校验和服务端合规扫描。任一步失败都不返回半成品。

## 工作流二：封面图制作

封面图是独立导航入口，与笔记生成没有图片依赖关系。

输入：

1. 业务类型。
2. 必选场景。
3. 一句案例简介。
4. 1–12 张活动原图。

### 生成封面提示词

用户点击“生成封面提示词”后，系统调用 `gpt-5.5` 的视觉能力：

- 分析每张图片的主体、互动动作、背景复杂度、方向和封面适配度。
- 选择最适合作为首图的图片并说明依据。
- 结合客户目的生成 3 条封面大字，推荐其中 1 条。
- 生成可编辑的正向提示词和限制条件。
- 提示词必须要求保留真实人物、设备和现场结构，不生成二维码、联系方式、额外 Logo 或虚假活动场景。

提示词结果只有真实接口成功后才展示。

### 生成封面图

用户确认或编辑提示词与封面大字后，点击“生成封面图”：

1. 服务端把 AI 推荐或用户选择的原图发送到 `/images/edits`。
2. 模型使用 `gpt-image-2`，默认中等质量，目标尺寸为 3:4 竖图。
3. 模型负责清理背景、调整光线、强化主体和活动氛围。
4. 为避免图片模型生成错误中文，模型输出画面不直接写标题；服务端使用 Sharp 将用户确认的封面大字准确叠加到最终图。
5. 输出文件保留 30 分钟，支持预览、重新生成和下载。

目标尺寸优先使用 `1152x1536`。如果中转站拒绝自定义 3:4 尺寸，改用其支持的竖版尺寸，并在服务端安全裁剪到 3:4。

## 工作流三：账号分析

账号分析必须使用 `gpt-5.5` 多模态接口读取真实截图。输入支持主页、笔记列表、单篇笔记和创作中心数据截图。

输出必须包含：

- 截图中可见事实。
- 对应截图编号和可见证据。
- 账号定位、简介、封面、标题、内容结构与合规诊断。
- 每条结论的置信度。
- 信息不足或看不清的项目。
- 按优先级排序的修改建议和 14 天行动计划。

模型不得编造截图中不存在的数据。服务端会拒绝缺少证据字段的分析结果。

## 工作流四：同行分析

同行分析使用 `gpt-5.5` 读取用户上传的封面、正文、主页和数据截图，并结合手动输入的标题、正文和互动数据。

输出必须包含：

- 标题、封面、正文结构及其可见依据。
- 点击、收藏或传播原因的事实与推断区分。
- 吸引的是精准客户还是泛流量。
- 可借鉴和不能照抄的部分。
- 按用户选择的业务与场景改写的 3 个标题和 1 个正文角度。
- 合规、版权、夸张承诺和客户隐私风险。

如果截图或文本不足以支持判断，返回“信息不足”而不是补写模板结论。

## 管理员接口配置

新增“接口配置”页面，并使用独立管理员密码解锁。普通同事只能使用业务功能。

配置项：

- API Base URL。
- API Key。
- 文本/视觉模型。
- 图片模型。
- 请求超时。
- 启用状态。

页面支持：

- 检测中转站鉴权和 `/models` 可访问性。
- 发送最小文本请求验证 `gpt-5.5`。
- 检查配置的图片模型是否在模型列表中。
- 显示最近一次测试结果和清晰错误。

保存后前端只显示 API Key 掩码，后端不会返回完整 Key。

### 配置存储

项目继续不使用数据库。API 配置使用 AES-256-GCM 加密后保存到服务器本地配置文件。加密密钥由服务器环境变量 `CONFIG_ENCRYPTION_KEY` 提供，管理员密码由 `ADMIN_PASSWORD` 提供。两者只在服务器 `.env` 中设置。

环境变量配置优先于加密文件，便于云服务器运维。业务历史、文案、截图和分析结果仍然不持久化。

## 后端边界

新增或调整以下接口：

- `POST /api/generate-note`：JSON 输入，不接收图片。
- `POST /api/generate-cover-prompt`：多图上传并调用视觉模型。
- `POST /api/generate-cover-image`：接收选中原图、提示词和封面大字并调用图片编辑接口。
- `POST /api/analyze-account`：真实多模态账号分析。
- `POST /api/analyze-competitor`：真实多模态同行分析。
- `POST /api/admin/login`：管理员鉴权。
- `GET /api/admin/ai-config`：返回脱敏配置和状态。
- `PUT /api/admin/ai-config`：校验并保存加密配置。
- `POST /api/admin/ai-config/test`：测试文本、模型列表和鉴权。

AI 调用统一经过 `LlmHubProvider`。路由、提示词、配置存储和第三方协议解析分开实现，避免页面直接依赖中转站响应格式。

## 错误处理

- 未配置 API：提示进入接口配置，不生成模板结果。
- 鉴权失败：提示检查中转站 Key。
- 模型不存在：显示实际模型字符串并提示在设置页修改。
- 响应不是合法 JSON：自动进行一次结构修复请求；再次失败则返回明确错误。
- 图片生成时间较长：显示最长约两分钟的进度说明，禁止重复提交。
- 中转站返回 URL 或 base64 图片时均能处理。
- 用户输入和上传文件在失败后保留在浏览器，便于重试。
- 服务端日志不得记录 API Key、完整图片 base64 或用户正文全文。

## 安全与隐私

- API Key 不进入前端源码、URL、浏览器存储或日志。
- 管理员令牌与普通使用令牌使用不同权限范围。
- Base URL 只允许 HTTPS，并默认限制为管理员配置的中转站域名，避免服务端请求伪造。
- 图片格式、大小、数量继续进行前后端双重验证。
- 上传图片在请求结束后删除；生成封面保留 30 分钟。
- 页面刷新后文案和分析结果清空。

## 测试与验收

自动测试使用本地 mock HTTP 服务模拟中转站，不调用真实 Key、不产生费用。

- 配置加密、掩码、管理员权限和 Base URL 校验。
- OpenAI 兼容文本、视觉、模型列表、图片编辑、base64 与 URL 返回解析。
- 九个场景的提示词隔离与客户目的差异。
- 笔记结果严格为 3 个标题、1 篇正文和正文末尾 8–12 个话题。
- Humanizer 与合规二次复核流程。
- 无 API 时不返回 Demo 内容。
- 账号和同行结果必须包含证据、置信度和信息不足项。
- 封面提示词与封面图按钮独立调用。
- 桌面端与手机端浏览器验收。

## 不在本次范围

- 自动发布、登录或爬取小红书。
- 文案、截图和分析历史记录。
- 数据库、会员、支付和复杂多用户权限。
- 文章内配图批量生成。
- 对点击量、咨询量或成交结果作保证。
