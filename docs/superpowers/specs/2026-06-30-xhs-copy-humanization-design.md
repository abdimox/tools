# 小红书笔记自然化与客户视角优化设计

## 目标

优化“生成笔记”的标题和正文，让内容更像熟悉活动现场的人在分享经验，减少模板腔和 AI 痕迹，同时继续服务于点击、咨询和合作转化。输出格式保持 3 个标题、1 篇正文，话题放在正文末尾。

## 不变约束

- 继续使用现有两次文字模型调用，不增加第三次调用和等待时间。
- DIY 与 Photobooth 业务不能混写。
- 企业、商场、楼盘、社区、4S 店、婚礼等场景继续使用各自的决策人和客户动机。
- 不虚构公司、城市、人数、时长、现场反应、客户评价、成交、客流或活动效果。
- 只有用户明确输入的案例事实才能写成已经发生的经历。
- 不输出诊断过程、评分表或修改说明，只输出可以直接复制的最终笔记。

## 两阶段生成流程

### 第一阶段：策略初稿

模型先在内部确定：

1. 谁是本场景的决策人。
2. 对方为什么现在需要这类活动。
3. 对方最担心的执行问题。
4. 什么具体信息值得点击。
5. 用户输入中有哪些事实可以使用。

初稿按“真实场景或现实顾虑 → 现场方案 → 对客户的实际意义”组织，但不强制每篇固定三段，也不使用统一开头。

### 第二阶段：内部诊断与重写

终审模型先对初稿内部评分，再直接返回重写后的最终稿。评分维度为：

- 标题点击理由
- 客户视角深度
- 事实可信度
- AI 感与语言自然度
- 关键词布局
- 话题相关性

任一核心项不合格时必须重写，不能只做同义词替换。评分只用于模型内部判断，不出现在页面结果中。

## 标题规则

- 严格输出 3 个角度不同的标题。
- 标题优先包含场景、客户问题或具体活动类型，兼顾搜索词和点击理由。
- 不机械套用“被夸爆、直接封神、闭眼冲、氛围拉满、疯狂拍照”等夸张词。
- 用户没有提供结果证据时，不写“员工都抢着参加、领导很满意、客户停留两小时”等结果。
- 三个标题不能只替换几个词。

## 正文规则

- 开头直接进入客户会遇到的情况，避免“随着……发展”“为了进一步提升”“我们很专业”等铺垫。
- 从 HR、行政、商场运营、楼盘营销、门店负责人、新人或活动策划的实际顾虑出发。
- 写清楚方案为什么适合当前场景，优先使用用户提供的具体事实。
- 信息不足时使用经验判断表达，例如“这类活动通常更看重……”，不能伪装成真实案例复盘。
- 长短句自然变化，避免连续排比、功能清单、空泛总结和过多连接词。
- 不固定使用“说实话、最省心的是、最后、总之”等模板过渡。
- 结尾给出克制的选择判断，不使用导流词或效果保证。

## 关键词与话题

- 关键词自然分布在标题、开头和方案说明中，不堆砌同义词。
- 话题保持 8–12 个，覆盖地域（仅在用户提供时）、业务、场景、活动类型和客户角色。
- 话题直接附在正文末尾，便于整段复制。

## 验证

- 提示词测试检查内部评分、事实门禁、自然表达和禁用模板词规则存在。
- 使用企业、商场、楼盘和 Photobooth 场景样例检查决策人角度不同。
- 保持现有 JSON 结构，确保前端无需改动。
- 运行全部测试、类型检查和生产构建后再部署。
