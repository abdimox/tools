# 文字与图片 API Key 分离设计

## 目标

同一中转地址 `https://llmhub.ltd/v1` 使用两个独立密钥：文字与视觉分析使用文字 API Key，封面生成使用图片 API Key。避免图片专用 Key 被用于文字接口后产生误导性的鉴权失败。

## 配置模型

现有单一 `apiKey` 拆为：

- `textApiKey`：用于 `gpt-5.5` 的文案、封面提示词、活动图分析、账号分析和同行分析。
- `imageApiKey`：仅用于 `gpt-image-2` 的封面图片生成或编辑。

两个密钥共享 Base URL、超时和启用状态，均使用 AES-256-GCM 加密保存，状态接口只返回各自的脱敏值。旧加密配置中的 `apiKey` 在读取时自动迁移为两个相同密钥，避免升级后立即失效；下次保存时写入新格式。

## 调用与测试

文字请求的 Authorization 使用 `textApiKey`，图片请求使用 `imageApiKey`，不得回退或互相替代。

设置页提供两个独立测试按钮：

- “测试文字接口”验证文字 Key、`gpt-5.5` 和中转站文字链路。
- “测试图片接口”验证图片 Key、`gpt-image-2` 和中转站图片鉴权链路。

测试结果分别显示，避免其中一项失败掩盖另一项状态。鉴权错误明确指出是文字 Key 还是图片 Key；未填写对应密钥时仅提示该字段，不影响保存另一字段的已有值。

## 界面

接口配置页将单一 API Key 输入框替换为“文字/视觉 API Key”和“图片 API Key”。每个字段独立显示/隐藏，已配置时显示各自脱敏提示。安全说明同步说明两个密钥均只提交到本机后端并加密保存。

## 验证

- 单元测试覆盖加密存取、旧配置迁移和两个密钥的请求路由。
- API 测试覆盖独立测试接口、保存时留空保留旧密钥、缺失密钥和鉴权错误文案。
- 浏览器回归覆盖管理员解锁、双密钥字段、独立测试按钮及移动端布局。
- 自动测试使用模拟中转站，不调用真实密钥或产生费用。
