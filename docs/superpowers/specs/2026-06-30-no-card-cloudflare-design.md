# 无银行卡 Cloudflare 部署调整

## 目标

在不购买服务器、域名或启用付费产品的前提下，让内部员工可以长期访问工作台。保留现有登录、员工隔离、AI 配置、GPT 对话和封面生成功能。

## 架构

- 使用 Cloudflare Worker 同时提供 API 和前端静态资源，访问地址使用免费的 `workers.dev` 子域名。
- 使用 Cloudflare D1 保存账号、会话、对话、消息、加密后的 AI 配置和压缩后的聊天图片。
- 不启用 R2。聊天图片在浏览器端压缩到 1.2 MB 以内后上传；Worker 再做大小和格式校验。
- 封面参考图只在当前请求中发送给图片模型；生成结果直接返回浏览器，不在 Cloudflare 保存。

## 数据与限制

`message_attachments` 增加 `blob_data BLOB`。附件仍由消息、对话和员工所有权约束，删除对话时由外键级联删除。单张持久化图片上限为 1.2 MB，每条消息最多四张。D1 空间有限，因此这是适合少量内部员工的低成本方案，不是图片素材库。

## 错误处理

- 浏览器压缩失败时不上传原始大图，并显示明确错误。
- Worker 拒绝超过持久化上限、格式不符或跨员工访问的图片。
- 封面生成失败时不留下临时文件；成功后由用户立即预览或保存。

## 验证

- D1 迁移可在本地和远端重复执行。
- 后端集成测试覆盖登录隔离和 D1 图片读取。
- 前后端类型检查、测试、生产构建及 Wrangler 部署均需通过。
